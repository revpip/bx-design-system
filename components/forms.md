# Form Components

Forms should feel like guided conversations, not admin paperwork.

## Principles

- Ask only what is needed at that stage.
- Use helper text to reduce uncertainty.
- Use progress indicators for multi-step forms.
- Make errors clear and kind.
- Preserve user input where possible.

## Field Standards

Every field should have:

- visible label
- optional helper text
- clear error state
- accessible focus state

## Assessment Forms

Assessment questions should:

- feel consultative
- avoid jargon
- explain why the question matters where useful
- support branching in future versions

## Error Copy

Bad:

> Invalid input.

Better:

> Please enter a valid email address so we can send your Business X-Ray summary.
