# Button Components

Buttons should make the next action obvious.

## Types

### Primary

Used for the main conversion or continuation action.

Examples:

- Start your Business X-Ray
- Book a confidential call
- Generate report
- Save settings

### Secondary

Used for supporting actions.

Examples:

- View report preview
- Download CSV
- Read how it works

### Ghost

Used for lower-emphasis navigation or contextual actions.

### Danger

Used only for destructive actions.

## Rules

- Every button label should describe the action.
- Avoid vague labels such as `Submit` or `Click here`.
- Never place two primary buttons side by side unless the choice is truly equal.
- Buttons should have generous padding and clear focus states.
- Button interactions should feel calm, not flashy.
