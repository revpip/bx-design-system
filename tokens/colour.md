# Colour Tokens

## Philosophy

The colour system is restrained, premium and functional. Colour should communicate meaning, not decorate unnecessarily.

## Core Palette

| Token | Hex | Purpose |
|---|---:|---|
| `midnight` | `#0F172A` | Primary background |
| `ink` | `#111827` | Panels and cards |
| `deep-space` | `#020617` | High-depth backgrounds |
| `gold` | `#D4AF37` | Signature accent / calls to action |
| `mist` | `#E5E7EB` | Primary text on dark |
| `muted` | `#94A3B8` | Secondary text |
| `white` | `#FFFFFF` | High emphasis |

## Semantic Palette

| Token | Hex | Purpose |
|---|---:|---|
| `success` | `#10B981` | Positive status / completed |
| `warning` | `#F59E0B` | Needs attention |
| `danger` | `#EF4444` | Critical risk |
| `info` | `#38BDF8` | Informational |

## Usage Rules

- Gold is an accent, not a background for large areas.
- Red is only for genuine risk or destructive actions.
- Never rely on colour alone to communicate status.
- Maintain WCAG AA contrast wherever possible.
